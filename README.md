**Atividade Prática**: Programação Orientada a Objetos
**Objetivo**: Aplicar os conceitos fundamentais de POO utilizando a linguagem C# e a plataforma .NET, focando em integridade de dados e lógica de negócio.
**Conceitos Aplicados (O que eu aprendi/usei):**
Encapsulamento: Proteção de dados sensíveis (como saldos e senhas) usando modificadores private, garantindo que o estado do objeto só seja alterado por métodos autorizados.
Construtores: Inicialização segura dos objetos, definindo estados iniciais obrigatórios (como a marca de uma lâmpada ou a classe de um guerreiro).
Campos Somente Leitura (readonly): Implementação de atributos imutáveis após a criação, como o número de uma conta bancária ou a tecnologia de uma lâmpada.
Campos Calculados: Uso de propriedades inteligentes que não ocupam espaço extra na memória, mas retornam valores baseados em outros atributos (ex: SaldoTotal ou StatusConta).
Sobrescrita de Métodos (override): Customização do método ToString() para que o objeto saiba se "apresentar" de forma clara e legível no console.
Lógica de Negócio e Validação: Implementação de travas de segurança, como bloquear um cofre após 3 erros ou impedir que um personagem morto receba cura.
**Tudo isso foram requisitos dos modulos.**

📂 Módulos do Projeto
**Lâmpada Inteligente:** Controle de estado e validação de brilho.
**Cofre Digital:** Sistema de segurança com bloqueio após 3 tentativas erradas.
**Conta Corrente Universitária:** Gestão de saldo e limite de cheque especial.
**Personagem de RPG:** Lógica de evolução de nível e estados de vida/morte.
